STAGE 1: REAL-TIME DETECTOR (transit_detector.py)
───────────────────────────────────────────────────────────────────────────────

Method: Dual-Signal Consecutive-Frame Confirmation
- Signal A: Frame-to-frame difference (detects motion)
- Signal B: Centre-weighted reference difference (detects position change)

CRITICAL FALSE NEGATIVE SOURCES:

1. WARMUP GATE (Line 449)
   └─ First 3 seconds: ZERO detections possible
   └─ Risk: Fast transits (ISS, satellites) in first 3s completely missed
   └─ Status: HARDCODED, not configurable

2. SPATIAL CENTRE-EDGE RATIO FILTER (Line 460)
   └─ Requirement: centre_score / edge_score ≥ 1.5
   └─ Risk: Edge transits (crossing frame boundary) rejected
   └─ Risk: Small objects with noisy edges randomly rejected
   └─ Status: HARDCODED (CENTRE_EDGE_RATIO_MIN=1.5), not directly configurable

3. CONSECUTIVE FRAMES REQUIREMENT (Lines 463-476)
   └─ Requirement: 3 consecutive frames above threshold
   └─ Risk: Very fast transits (1-2 frame duration) completely rejected
   └─ Risk: Jittery objects toggle threshold → counter resets
   └─ Status: HARDCODED (CONSEC_FRAMES_REQUIRED=3), not configurable

4. ADAPTIVE THRESHOLDING (Lines 479-488, 453-454)
   └─ Formula: median + max(3×MAD, 0.5×median)
   └─ Risk: Faint objects (birds, distant aircraft) below 3-sigma
   └─ Risk: High scintillation raises baseline → higher threshold
   └─ Status: Configurable via sensitivity_scale (default 1.0)
   └─ Tuning: Increase sensitivity_scale to 1.5-2.0 for fainter objects

5. 30-SECOND COOLDOWN (Line 40, 470)
   └─ Requirement: At least 30s between detections
   └─ Risk: Two real transits 20s apart → second one IGNORED
   └─ Status: HARDCODED, not configurable

TUNABLE PARAMETERS SUMMARY
───────────────────────────────────────────────────────────────────────────────

DETECTOR (Real-Time):
  • sensitivity_scale: [default 1.0] → increase to 1.5-2.0 for fainter objects
    └─ Multiplies adaptive thresholds (lower = more sensitive)

ANALYZER (Post-Capture):
Via analyze_video() API:
  
  • min_blob_pixels: [default 20] → reduce to 5-10 for small objects
    └─ Minimum silhouette size in pixels
  
  • diff_threshold: [default 15] → reduce to 8-10 for low-contrast
    └─ Intensity difference threshold for object detection
  
  • disk_margin_pct: [default 0.12] → reduce to 0.02-0.05 for limb transits
    └─ Fraction of disk radius to exclude from detection area
  
  • target: ["auto", "moon", "sun"] → affects min_speed_px_s, min_travel_px
    └─ "moon" → 40px/s min (vs 80 for sun)
    └─ "moon" → 20px min travel (vs 40 for sun)
